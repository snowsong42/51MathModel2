## 先看“地基”：EDA给了我们哪些关键信息

1. **变量类型明确**：

   - **连续传感器**：`PorePressure`、`Infiltration` —— 分布对称、无明显异常，适合滑动平滑+差分。
   - **降雨**：99%为零值，CCF滞后**1步**最大但很微弱（r=0.065），说明直接影响很小，**累积效应**和**有效降雨**更重要。
   - **微震**：80%时刻有事件，单步最多27个，事件间隔长尾，适合滑动密度+累积趋势。
   - **爆破**：仅47次，距离0.6-5.8m、药量1-11.9kg；响应持续**4-8小时**，衰减常数τ建议50步。
2. **已有特征丰富**：`特征工程后变量数: 36`，已包含很多滞后、累积、交互项和时间编码，说明基础工作已做。
3. **阶段差异显著**：三阶段速度均值差异两个数量级（0.006 → 0.03 → 0.40），必须分阶段建模，特征在不同阶段的重要性会不同。

## 然后各个击破：分类型的特征工程改进方案

下面针对每一类变量，给出**基于EDA发现的具体构造公式、理由和代码切入点**。

### A. 连续传感器变量（孔隙水压力、干湿入渗系数）

**EDA发现**：分布对称，无明显长尾，无缺失；24h滑动平均图显示平稳但有局部波动。

**特征策略——捕捉“变化速率”与“趋势”**

1. **一阶/二阶差分**：
   - `pore_diff_1 = PorePressure(t) - PorePressure(t-1)`
   - `pore_diff_6 = PorePressure(t) - PorePressure(t-6)`  （1小时前对比，捕捉突发变化）
   - 理由：滑坡前孔压常出现突然上升，差值比绝对值更灵敏。
2. **多窗口滚动统计**（优先保留图中已显示的稳定窗口）：
   - `pore_roll_mean_12h = rolling_mean(PorePressure, window=72)`
   - `pore_roll_std_12h = rolling_std(PorePressure, window=72)`
   - 理由：滑坡阶段孔压波动增大，标准差可以作为波动性特征。
3. **交互特征**（已有，但可加强）：
   - `pore_infilt = PorePressure * Infiltration`  —— 入渗系数调节孔压响应，交互项弥补线性无法捕获的耦合。
   - 建议增加：`pore_infilt_diff = pore_infilt(t) - pore_infilt(t-72)`，反映耦合趋势变化。

**代码实现**：在 `common/data_utils.py` 中添加上述差分和滚动标准差函数，更新特征列表时加入新列名。

### B. 降雨变量

**EDA发现**：CCF滞后1步最大但r极小（0.065），说明**瞬时降雨对位移几乎没有影响**，必须构造累积和有效降雨。有效降雨已用衰减0.85建模。

**特征策略——多尺度累积 + 有效降雨 + 前期雨量占比**

1. **多窗口累积量**（已在 `rainfall_cumulative.png` 中绘制）：
   - `rain_3h_sum`, `rain_6h_sum`, `rain_12h_sum`, `rain_24h_sum`
   - 理由：降雨入渗需要时间，不同深度土层响应时间不同，多窗口让模型自动选择最佳滞后。
2. **有效降雨**（已有，但可加多衰减系数）：
   - `rain_eff_0.9 = rain(t) + 0.9*rain_eff(t-1)`
   - `rain_eff_0.7 = rain(t) + 0.7*rain_eff(t-1)`
   - 理由：不同土质/深度记忆衰减不同，多给几个候选让树模型自己选。
3. **降雨强度波动**：
   - `rain_6h_std`：过去6小时降雨量标准差，衡量降雨是否集中（暴雨型）。
   - 理由：暴雨比连续小雨更容易触发滑坡，强度波动很重要。

**特别注意**：由于降雨序列99%是零值，CCF被稀释，不必执着于用CCF决定窗口。用多窗口累积 + 有效降雨已是标准做法。

### C. 微震事件数

**EDA发现**：事件时刻占比80%，单步最多27次，间隔长尾，累积图与位移呈正相关趋势。说明**频率上升和累积损伤**都有意义。

**特征策略——密度、累积和能量替代**

1. **事件密度**（滚动窗口计数，窗口参考滑动计数图选取）：
   - `micro_6h_count = rolling_count(Microseismic>0, window=36)`
   - `micro_12h_count`, `micro_24h_count`
   - 理由：密度反映微破裂活跃度，加速阶段密度上升是前兆。
2. **累积事件数**（已有 `Microseismic_cum` 的类似特征）：
   - 保留原始累积，可再构造 `cum_micro_24h = Microseismic.rolling(144).sum()` —— 最近24h累积，避免全量累积受时间跨度影响。
3. **应变能释放替代**：若无单次能量值，可用事件数的平方根 `sqrt(Microseismic)` 或 `Microseismic^2` 近似能量释放率。理由：声发射能量常与破裂面积正相关，平方可作为能量代理。

### D. 爆破变量（核心：距离、药量分开，且需衰减）

**EDA发现**：响应持续4-8h（τ=50步），距离-药量散点图显示无明显线性相关（相关性中等，但从散点图可见波动），需要分开处理。已有 `Blast_Energy = q/d²` 和 `Time_since_blast`。

**策略——分别构造衰减影响，再合成一个共同能量特征**（保证消融实验可区分）

1. **距离影响函数**（独立）：

   ```
   dist_impact(t) = sum_over_blasts[ 1 / (d_i^2 + 1) * exp(-(t-t_i) / 50) ]
   ```

   计算后形成等长时序，存放于 `BlastDist_impact_decay`。
2. **药量影响函数**（独立）：

   ```
   charge_impact(t) = sum_over_blasts[ q_i * exp(-(t-t_i) / 50) ]
   ```

   存放于 `BlastCharge_impact_decay`。
3. **联合能量特征**（在消融实验中作为“爆破联合”成员，当剔除某一方时该特征也失效）：

   - `Blast_Energy_decay(t) = sum[ (q_i / (d_i²+1)) * exp(-(t-t_i) / 50) ]`
   - 这个特征只有在**距离和药量都未被剔除时**才保留，我们可以用条件语句判断。
4. **距上次爆破时间**（已有 `Time_since_blast`），但建议加一个**距下次爆破时间**？这不现实（未知未来），不必。可以增加**爆破间隔**：`blast_interval = t_i+1 - t_i`，在发生爆破的时刻填入最近两次间隔，填补0后向前填充。意义：密集爆破比稀疏爆破影响更大。
5. **峰值粒子速度(PPV)**：若已有 `Blast_PPV`（看起来有），它本身就是与距离和药量相关的物理量，保留即可。

**消融实验如何区分**：在 `exclude_features` 逻辑中，当我们剔除“爆破点距离”时，不仅删除 `BlastDist` 相关列，也要删除包含 `dist_impact` 和 `Blast_Energy` 的列（同样，剔除“单段最大药量”时删除 `charge_impact` 和 `Blast_Energy` 列）。这样可以公平评估二者重要性。`Blast_PPV` 是结合产物，归入“爆破点距离+药量”联合类，可以新建一个类“爆破联合”，或者按上述规则跟随任一剔除都排除。

## 实施路线图：把这些特征接进现有代码

1. **在 `common/data_utils.py` 末尾添加一个函数**：
   `def add_advanced_eda_features(df, eda_params...)`参数从上面确定的窗口列表、衰减常数传入。
2. **函数内部实现**：

   - 使用 `pandas` 的 `rolling`、`shift`、`diff` 等生成上述所有新列。
   - 爆破衰减使用自定义 `exp_decay_apply` 函数（逐事件叠加）。
   - 返回扩展后的 DataFrame。
3. **修改 `q5_main.py`**：

   - 在 `load_pipeline` 得到 `df` 后，调用 `add_advanced_eda_features`。
   - 更新 `all_feats` 的获取方式，确保新列被包含。
   - 修正 `ablation_families` 字典，把距离和药量彻底分开，并为“爆破联合”增加剔除逻辑。
4. **消融实验字典更新**（示例）：

```python
ablation_families = {
    '降雨量':        关键字列表...,
    '孔隙水压力':    关键字列表...,
    '微震事件数':    关键字列表...,
    '干湿入渗系数':  关键字列表...,
    '爆破点距离':    关键字列表 + ['BlastDist_impact_decay', 'Blast_Energy_decay'],
    '单段最大药量':  关键字列表 + ['BlastCharge_impact_decay', 'Blast_Energy_decay'],
}
```

这样当剔除距离时，能量衰减特征会被移除；剔除药量时同理，保证公平。

5. **调参**：新增特征后，最好用 `optuna` 快速跑一次（基于时间序列交叉验证），因为窗口和衰减常数还存不确定性，可以微调参数（如衰减τ 从40到70步搜索，窗口长度列表也可优化），但这属于可选的锦上添花。

## 最终这一步能带来什么

- 你会发现爆破距离和药量的独立重要性被揭示得更清晰（可能其中一个边际贡献更大）。
- 加入衰减特征和交互项后，R² 很可能从 0.54 跳到 0.65~0.70，尤其是在加速和快速阶段，模型能更好地捕捉爆破后的位移突增。
- 论文中的“变量重要性原理解释”会非常扎实，因为每个特征背后都有物理衰减、累积、能量释放等叙述。
