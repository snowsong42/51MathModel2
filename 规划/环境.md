# 五一数学建模 Python 环境技术说明

本文档概述本项目通过 `uv` 管理的 Python 虚拟环境所包含的主要软件包，并按功能领域进行分类说明，供团队建模、编程及论文撰写时参考。所有依赖均已锁定在 `uv.lock` 中，保证成员间环境一致。

## 1. 核心数值计算与科学计算
- **NumPy** – 提供多维数组对象及丰富的数学函数，是整个科学计算生态的基础。
- **SciPy** – 基于 NumPy 的科学计算包，涵盖优化、插值、积分、信号处理、统计分布、线性代数等常用算法。
- **pandas** – 面向结构化数据的分析库，提供 DataFrame 等数据结构，用于清洗、变换、聚合与探索表格型数据。

## 2. 符号计算与数学推理
- **SymPy** – 纯 Python 符号数学库，支持代数化简、方程求解、微积分、极限、矩阵符号运算等解析推导。
- **mpmath** – 高精度浮点算术库，为 SymPy 提供高阶数值支持，也可独立完成任意精度计算。

## 3. 概率统计与计量经济学
- **statsmodels** – 统计模型估计与推断库，提供经典线性回归、广义线性模型、时间序列分析（ARIMA、VAR）、假设检验等工具。
- **patsy** – 描述统计模型公式的库，允许使用 R 风格公式语法来构建设计矩阵，常与 statsmodels 搭配。

## 4. 优化与运筹学
- **OR-Tools** – Google 开源的运筹优化工具包，包含约束编程（CP-SAT）、整数规划、线性规划、车辆路径等求解引擎。
- SciPy 的 `optimize` 模块也可处理连续与部分离散优化问题，二者互补。

## 5. 网络与图论分析
- **NetworkX** – 用于创建、操作和分析复杂网络（图）的库，提供经典图算法、中心性度量、连通性分析及简单网络可视化。

## 6. 机器学习与深度学习
- **PyTorch** – 主流深度学习框架，支持张量计算、自动求导、GPU 加速，可构建与训练各类神经网络。
- **torchvision** – PyTorch 的计算机视觉扩展，包含常见数据集、图像预处理变换及预训练模型。
- **torchaudio** – PyTorch 的音频处理扩展，提供音频文件 I/O、特征提取（如 MFCC）和数据增强功能。

## 7. 可视化
- **Matplotlib** – 功能全面的 2D 绘图库，可生成出版级质量的图表，是其他可视化库的底层基础。
- **Seaborn** – 基于 Matplotlib 的高级统计可视化库，提供美观默认样式和便捷的统计图表（分布图、回归图等）。
- **matplotlib-venn** – 绘制维恩图的 Matplotlib 扩展。
- **Pillow** – Python 图像处理库，支持读写、转换、简单过滤等操作，Matplotlib 依赖其读取图像。
- 底层支撑组件：`kiwisolver`（几何布局）、`contourpy`（等值线计算）、`cycler`（样式循环）、`fonttools`（字体渲染）等。

## 8. 交互式开发环境（Jupyter 生态）
- **JupyterLab** – 新一代交互式开发环境，集成 Notebook、代码控制台、终端等界面。
- **Notebook** – 经典的基于浏览器的交互式笔记本服务。
- **IPython** – 增强的交互式 Python 解释器，提供语法高亮、自动补全、魔法命令等功能。
- **ipykernel** – IPython 内核，供 Jupyter 客户端连接并执行代码。
- 辅助组件：`jupyter-client`（通信）、`jupyter-core`（核心接口）、`jupyter-server`（服务器）、`nbformat`（笔记本格式）、`nbconvert`（格式转换，如导出 PDF）、`nbclient`（执行笔记本）等。
- 交互控件与调试：`ipywidgets`（交互式控件）、`debugpy`（远程调试支持）。

## 9. 网络通信与数据获取
- **requests** – 简洁易用的 HTTP 客户端，用于访问网络 API、下载数据等。
- **httpx** – 支持异步的现代 HTTP 客户端，可作为 requests 的替代。
- 底层依赖：`urllib3`（HTTP 连接池）、`certifi`（CA 证书）、`charset-normalizer`（编码检测）、`idna`（国际化域名编码）、`websocket-client`（WebSocket 支持）。

## 10. 通用工具与重要依赖
- **tqdm** – 快速、可扩展的进度条，便于循环处理时展示进度。
- **PyYAML** – YAML 格式解析与生成，常用于读写配置文件。
- **psutil** – 跨平台的系统监控与进程管理库。
- **protobuf** – Google 的序列化结构数据格式，OR-Tools 等依赖于此。
- **absl-py** – Abseil 的 Python 公共库，被 OR-Tools 等 Google 项目内部使用。
- **typing-extensions** – 提供新版本类型注解的向后移植，保证类型标注兼容性。
- **markupsafe** / **Jinja2** – 模板引擎，用于生成文本（如 Jupyter 转换模板）。
- **beautifulsoup4** / **lxml** / **lark** / **mistune** – HTML/XML 解析、Markdown 处理等杂项文本处理。
- **pyzmq** / **tornado** – 异步网络库和消息队列，为 Jupyter 消息传递提供支撑。
- 其余实用库：`cffi`（外部 C 函数调用）、`pygments`（语法高亮）、`jsonschema`（JSON 结构校验）、`attrs`（简化类定义）、`bleach`（HTML 消毒）等。

## 使用与维护要点
- 所有依赖均通过 `uv` 管理，锁文件 `uv.lock` 记录精确版本和下载源，可保证团队环境完全一致。
- 在项目根目录执行 `uv sync` 即可安装或同步所有依赖。
- PyTorch 通过 PyTorch 官方 CUDA 12.8 (`cu128`) 索引安装，支持 GPU 加速；若无需 GPU，可移除 `extra-index-url` 后重新 `uv sync`，将自动安装 CPU 版本。
- 可使用随附的 `check_env.py` 快速验证核心包的导入和基本功能。
- 新增依赖时，务必使用 `uv add <package>` 命令，以便自动更新 `pyproject.toml` 和 `uv.lock`。
