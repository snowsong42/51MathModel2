# Q5.1 最优变量组合建模 — 算法分析与改进方案

## 一、当前算法概述

### 模型架构
- **底层模型**：Gradient Boosting Regression Tree (GBRT)
- **参数配置**：`n_estimators=200, learning_rate=0.01, max_depth=5, subsample=0.8`
- **目标变量**：10分钟位移增量 `ΔD`（一阶差分）
- **评估策略**：按时间顺序 80/20 划分（前80%训练，后20%测试）

### 特征体系
| 类别 | 特征 | 说明 |
|------|------|------|
| 原始6变量 | Rainfall, PorePressure, Microseismic, Infiltration, BlastDist, BlastCharge | 题目给定的6个特征 |
| 变化率 | Pore_Diff | 孔压一阶差分 |
| 24h累积 | Rain_cum24, Blast_E_cum24 | 滚动144步求和 |
| 时间衰减 | Time_since_blast, Time_since_rain | 距上次非零事件的时间步数 |

## 二、算法优势

### 2.1 GBRT 的天然优势
| 特性 | 优势说明 |
|------|----------|
| **捕捉非线性关系** | GBRT 通过多棵决策树的集成，可以自动学习特征与目标之间的非线性映射，无需像线性回归那样手动构造交互项 |
| **特征重要性内置** | 训练后可直接输出 `feature_importances_`，便于分析各变量对预测的贡献度 |
| **鲁棒性较好** | 对特征尺度不敏感（无需归一化）；对部分缺失值有一定容忍度 |
| **树模型适合表格数据** | 在结构化/表格数据上，GBRT 通常优于深度学习模型 |

### 2.2 特征工程合理性
| 设计 | 合理性 |
|------|--------|
| **差分目标 ΔD** | 将位移预测转为增量预测，消除长期趋势/单位根影响，使序列更平稳 |
| **24h滚动累积** | 降雨和爆破的影响具有滞后性，滚动和能够捕获累积效应（入渗需要时间传导到监测点） |
| **距上次事件时间** | 爆破和降雨是偶发事件，距上次事件的时间步可以反映"恢复/衰减"状态 |
| **孔压差分** | 孔压的变化率比绝对值更能反映水力梯度驱动的变形 |

### 2.3 对比策略合理性
- **逐一剔除法（Leave-One-Out）**：从6个原始特征中每次去掉1个，共形成6种组合。这种方法直接回答了"哪个特征最不重要"（去掉后RMSE下降=该特征冗余/有害；去掉后RMSE上升最多=该特征最重要）。

## 三、算法劣势与改进空间

### 3.1 数据划分问题 ⭐ 最重要

| 问题 | 描述 |
|------|------|
| **当前做法** | 简单的 80/20 时间顺序划分 |
| **问题** | 边坡变形具有**非平稳性**——训练集（前期）和测试集（后期）可能处于不同变形阶段。例如：训练集处于匀速段，测试集处于加速段，模型会严重漏报 |
| **改进方案** | ① **交叉验证+时间序列折**：使用 `TimeSeriesSplit` 而非随机 K-Fold；② **阶段感知划分**：按三阶段（匀速/加速/快速）分别划分，确保每个阶段都有训练和测试样本；③ **滚动预测窗口**：在时间序列上滑动预测窗口，每次用过去N天预测未来M步 |

### 3.2 目标变量选择问题

| 问题 | 描述 |
|------|------|
| **当前做法** | 预测 10分钟位移增量 ΔD |
| **问题** | ΔD 信噪比极低——短期变形增量往往淹没在噪声中（1.3mm 的 RMSE vs 总位移可能达数百 mm）。模型 R² ≈ 0.2 说明大部分方差未被解释 |
| **改进方案** | ① **多步预测**：改为预测未来 1h/6h/24h 累积位移，平滑噪声；② **位移水平预测**：直接预测位移绝对值（需保留趋势），可用差分+逆差分还原；③ **分阶段建模**：不同阶段影响规律不同，统一建模会稀释各阶段特异性 |

### 3.3 爆破特征处理粗糙

| 问题 | 描述 |
|------|------|
| **当前做法** | NaN → 0，再用滚动累积和距上次时间 |
| **问题** | 爆破的影响不应仅由药量和距离的简单乘积决定。爆破激发的应力波传播具有**方位角依赖**（爆心相对监测点的方向），简单的"距离倒数立方"衰减模型过于简化 |
| **改进方案** | ① **Peak Particle Velocity 模型**：`PPV = k·(Q/R)^α`，对每个爆破计算等效振动强度；② **分方向爆破当量**：如果知道爆心方位，可分解为顺坡和垂直方向的投影；③ **频域特征**：爆破产生的振动频率特征（低频=整体位移，高频=局部破裂） |

### 3.4 缺失滞后-响应时间

| 问题 | 描述 |
|------|------|
| **当前做法** | 仅用当前时刻的特征预测当前增量（同步回归） |
| **问题** | 降雨→入渗→孔压升高→变形存在**显著滞后**（可能数小时到数天）。同步回归会低估降雨和爆破的实际贡献 |
| **改进方案** | ① **时延互相关分析**：先计算每个特征与位移的互相关函数，确定最优滞后步数；② **LSTM/GRU 等时序模型**：直接建模输入序列到输出序列的映射，自然包含滞后关系；③ **特征滞后窗**：将过去 N 步的特征拼接到当前特征向量中 |

### 3.5 GBRT 超参数未调优

| 问题 | 描述 |
|------|------|
| **当前做法** | 固定参数 `n_estimators=200, lr=0.01, max_depth=5` |
| **问题** | 未做超参数搜索，可能导致欠拟合或过拟合 |
| **改进方案** | 使用 `GridSearchCV` 或 `Optuna` 搜索：`n_estimators∈[100,500]`, `max_depth∈[3,7]`, `lr∈[0.005,0.05]`, `subsample∈[0.6,1.0]` |

### 3.6 缺乏不确定性量化

| 问题 | 描述 |
|------|------|
| **当前做法** | 仅输出点预测（单值） |
| **问题** | 对于预警任务，知道"预测值 ± 不确定性区间"比点预测更重要——高不确定性时应降低预警置信度 |
| **改进方案** | ① **分位数回归森林（Quantile Regression Forest）**：输出预测分位数；② **Bayesian GBRT（如 PGBM 的 Bayesian 模式）**；③ **Dropout 近似贝叶斯**：训练时随机丢弃子树，推理时多次前传得到分布 |

### 3.7 变量组合枚举不完备

| 问题 | 描述 |
|------|------|
| **当前做法** | 6个变量中枚举所有 5-变量组合（6种） |
| **问题** | ① 题目要求的是"从6个中选5个"，但衍生特征（滚动和、差分）也是信息——去掉原始变量但保留其衍生特征会导致信息泄露；② 只枚举了"去掉1个"，未考虑"去掉2个"或不同顺序的逐步特征选择 |
| **改进方案** | ① **严格消融实验**：去掉某变量后，其所有衍生特征也应一并移除（目前已部分实现但可能不彻底）；② **前向/后向逐步选择**：从空集开始逐个添加、或从全集开始逐个删除，选择最优组合；③ **SHAP 值分析**：不依赖组合枚举，直接计算每个特征对预测的贡献量 |

### 3.8 模型单一

| 问题 | 描述 |
|------|------|
| **当前做法** | 仅使用 GBRT |
| **问题** | 单一模型方差高，且不同模型可能捕获数据的不同模式 |
| **改进方案** | ① **Stacking 集成**：GBRT + Random Forest + XGBoost + 线性模型作为基学习器，Linear Regression 作为元学习器；② **加权平均集成**：按验证集性能加权融合；③ **LightGBM / XGBoost / CatBoost** 替代 sklearn GBRT（训练更快、性能更优） |

### 5.2 滞后互相关分析 (P0)
```python
def find_optimal_lag(x, y, max_lag=288):
    """计算最优滞后步数（最多24小时）"""
    corrs = []
    for lag in range(max_lag + 1):
        if lag == 0:
            corr = np.corrcoef(x, y)[0, 1]
        else:
            corr = np.corrcoef(x[:-lag], y[lag:])[0, 1]
        corrs.append(corr)
    return np.argmax(np.abs(corrs)), np.max(np.abs(corrs))

# 对每个特征找到最优滞后步数，然后构造移位后的特征
lag_rain, _ = find_optimal_lag(df['Rainfall'], df['Delta_D'], max_lag=288)
df['Rainfall_lagged'] = df['Rainfall'].shift(lag_rain)
```

## 六、总结

当前 `test.py` 实现了 Q5.1 的基本要求（GBRT + 变量组合枚举），能够正常运行并输出对比结果。但它本质上是**同步回归模型**，没有充分利用时间序列的滞后动态特性，这是 R² 仅约 0.2 的根本原因。

**核心改进方向**：
1. **短期（P0-P1）**：加入滞后特征 + 时间序列 CV，R² 有望提升至 0.4-0.6
2. **中期（P1-P2）**：分阶段建模 + 超参数调优 + LightGBM 替代
3. **长期（P2-P3）**：不确定性量化 + SHAP 解释 + 多模型集成

对于边坡预警这类高价值决策任务，**知道"什么时候模型不可靠"往往比"得到一个点预测"更重要**——因此不确定性量化（3.6）虽然优先级为 P3，但在实际部署中价值最大。
